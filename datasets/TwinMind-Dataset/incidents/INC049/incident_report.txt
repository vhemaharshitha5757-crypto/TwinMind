1. SUMMARY
========================================================================

The Kong API Gateway (kong-prod-east-01) experienced intermittent 500 errors starting at 10:40 UTC on December 15, 2025. A custom Lua request transformation plugin crashed when processing requests containing Unicode characters in specific header fields. The plugin did not handle multi-byte character encoding properly, causing a nil reference error in the Lua runtime. Approximately 8% of API requests contained the triggering character pattern.

This incident was classified as P2 severity and was assigned to
Jenny Liu in the API Engineering department. The affected system was
Kong API Gateway, specifically the kong-prod-east-01 component located in US-East-1 (AWS).
The incident was detected through automated monitoring and alerting systems,
and the response team was engaged within minutes of the initial alert.

========================================================================

8% of API requests failing with 500 errors for 25 minutes. Affected requests were random based on content, making the issue difficult to reproduce. 2,800 API calls failed. 6. ROOT CAUSE ANALYSIS
========================================================================

The root cause of this incident was determined to be:
Custom Lua request transformation plugin did not handle multi-byte UTF-8 character encoding in HTTP header values, causing nil reference crash when processing Unicode characters.

Further analysis confirmed that this issue had been developing over time but
was not detected by existing monitoring thresholds and alerting rules. The
failure point was within the kong-prod-east-01 component of the Kong API Gateway
system. Contributing factors included insufficient monitoring coverage for
the specific failure mode encountered and the absence of automated
remediation procedures for this particular scenario.

========================================================================
7. RESOLUTION
========================================================================

The incident was resolved by the following actions: Deployed patched version of the transformation plugin with proper UTF-8 handling, enabled Unicode-safe string operations in Lua, and added input validation for header encoding.

The resolution was executed by Jenny Liu with support from the incident
response team. After applying the remediation steps, the team monitored the
system for a stabilization period to confirm that the issue was fully
resolved and no secondary or cascading failures had occurred. All affected
services were verified to be operating within normal parameters before the
incident was formally closed.

8. PREVENTIVE ACTIONS
========================================================================

The following preventive actions have been identified to reduce the
likelihood and impact of similar incidents in the future:

  1. Implement UTF-8 input handling testing for all custom Kong plugins
  2. Add plugin crash monitoring with automatic fallback to bypass mode
  3. Create internationalization test suite for API gateway plugins
  4. Implement Lua error handling best practices in plugin development guidelines
  5. Add canary testing with diverse character sets for plugin deployments

These action items will be tracked in the engineering backlog and prioritized
based on risk assessment and business impact analysis.
