1. SUMMARY
========================================================================

A production deployment via Jenkins pipeline (deploy-pipeline-prod-01) at 14:30 UTC on June 2, 2025 included a database migration that dropped a column still referenced by the previous application version. During the rolling deployment, instances running the old code began failing when querying the dropped column. The deployment was rolled back, but the database migration could not be simply reversed, requiring manual schema repair.

This incident was classified as P1 severity and was assigned to
Tom Williams in the CI/CD Engineering department. The affected system was
Jenkins CI/CD Pipeline, specifically the deploy-pipeline-prod-01 component located in US-East-1 (AWS).
The incident was detected through automated monitoring and alerting systems,
and the response team was engaged within minutes of the initial alert.

========================================================================
2. BUSINESS IMPACT
========================================================================

Production application partially down for 55 minutes during rollback and schema repair. 40% of application instances failed immediately after the migration ran. Approximately 15,000 users experienced errors. Data integrity was maintained but the incident exposed gaps in the deployment safety process.

The incident triggered the organization's major incident response process,
involving cross-functional coordination between engineering, operations, and
business stakeholders. Customer support was notified and prepared to handle

5. INVESTIGATION

Upon receiving the automated alert, the incident response team initiated a

Log analysis from the affected component showed error patterns that pointed

6. ROOT CAUSE ANALYSIS
========================================================================

The root cause of this incident was determined to be:
Database migration dropped a column before all application instances were updated to the new version that no longer references it, violating the expand-and-contract migration pattern.

Further analysis confirmed that this issue had been developing over time but
was not detected by existing monitoring thresholds and alerting rules. The
failure point was within the deploy-pipeline-prod-01 component of the Jenkins CI/CD Pipeline
system. Contributing factors included insufficient monitoring coverage for
the specific failure mode encountered and the absence of automated
remediation procedures for this particular scenario.

========================================================================
7. RESOLUTION
========================================================================

The incident was resolved by the following actions: Manually re-added the dropped column to the database, completed the application rollback, re-planned the migration using expand-and-contract pattern with separate deploy phases.

The resolution was executed by Tom Williams with support from the incident
response team. After applying the remediation steps, the team monitored the
system for a stabilization period to confirm that the issue was fully
resolved and no secondary or cascading failures had occurred. All affected
services were verified to be operating within normal parameters before the
incident was formally closed.

========================================================================
8. PREVENTIVE ACTIONS
========================================================================

The following preventive actions have been identified to reduce the
likelihood and impact of similar incidents in the future:

  1. Enforce expand-and-contract pattern for all breaking schema changes
  2. Implement database migration pre-check against running application versions
  3. Add deployment canary phase with automatic rollback triggers
  4. Create database migration rollback scripts for every migration
  5. Implement blue-green deployment for breaking changes

These action items will be tracked in the engineering backlog and prioritized
based on risk assessment and business impact analysis.
