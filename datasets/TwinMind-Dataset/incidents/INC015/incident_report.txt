1. SUMMARY
========================================================================

Nginx reverse proxy node nginx-prod-web-02 began returning 502 Bad Gateway errors for 30% of requests at 10:55 UTC on April 21, 2025. The issue was caused by ephemeral port exhaustion on the nginx node when communicating with upstream application servers. A keepalive misconfiguration was causing nginx to open a new connection for every request instead of reusing connections, rapidly consuming all available ephemeral ports.

This incident was classified as P2 severity and was assigned to
Alex Johnson in the SRE department. The affected system was
Nginx Reverse Proxy Cluster, specifically the nginx-prod-web-02 component located in US-West-2 (AWS).
The incident was detected through automated monitoring and alerting systems,
and the response team was engaged within minutes of the initial alert.

========================================================================
2. BUSINESS IMPACT
========================================================================

30% of web traffic served by nginx-prod-web-02 received 502 errors for 20 minutes. Approximately 6,000 users encountered error pages. Revenue impact estimated at $25,000 from abandoned shopping carts and failed transactions.

The incident triggered the organization's major incident response process,
involving cross-functional coordination between engineering, operations, and

6. ROOT CAUSE ANALYSIS
========================================================================

The root cause of this incident was determined to be:
Nginx keepalive configuration set to 0, disabling connection reuse to upstream servers, causing ephemeral port exhaustion during traffic spikes.

Further analysis confirmed that this issue had been developing over time but
was not detected by existing monitoring thresholds and alerting rules. The
failure point was within the nginx-prod-web-02 component of the Nginx Reverse Proxy Cluster
system. Contributing factors included insufficient monitoring coverage for
the specific failure mode encountered and the absence of automated
remediation procedures for this particular scenario.

========================================================================
7. RESOLUTION
========================================================================

The incident was resolved by the following actions: Corrected nginx keepalive configuration to maintain 256 persistent connections per upstream, restarted nginx to release TIME_WAIT sockets, and increased ephemeral port range.

The resolution was executed by Alex Johnson with support from the incident
response team. After applying the remediation steps, the team monitored the
system for a stabilization period to confirm that the issue was fully
resolved and no secondary or cascading failures had occurred. All affected
services were verified to be operating within normal parameters before the
incident was formally closed.
