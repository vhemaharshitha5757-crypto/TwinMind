1. SUMMARY
========================================================================

Azure Blob Storage account (stprodeufiles01) stopped accepting file uploads at 09:15 UTC on September 29, 2025. The SAS (Shared Access Signature) token used by the upload service had been regenerated with read-only permissions instead of read-write during a routine key rotation. All user file uploads, document attachments, and media uploads failed with AuthorizationPermissionMismatch errors.

This incident was classified as P2 severity and was assigned to
Lisa Park in the Cloud Infrastructure department. The affected system was
Azure Blob Storage, specifically the stprodeufiles01 component located in West Europe (Azure).
The incident was detected through automated monitoring and alerting systems,
and the response team was engaged within minutes of the initial alert.

File upload functionality unavailable for EU users for 30 minutes. 1,400 file upload attempts failed. Users unable to attach documents to support tickets, upload profile pictures, or submit forms with attachments. 280 support tickets created about upload failures.

========================================================================
8. PREVENTIVE ACTIONS
========================================================================

The following preventive actions have been identified to reduce the
likelihood and impact of similar incidents in the future:

  1. Implement SAS token permission validation in rotation automation
  2. Use managed identity instead of SAS tokens for service authentication
  3. Add write permission verification test after token rotation
  4. Create SAS token rotation runbook with permission checklist
  5. Implement automated upload smoke test after credential changes

These action items will be tracked in the engineering backlog and prioritized
based on risk assessment and business impact analysis.
